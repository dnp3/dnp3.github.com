enum opendnp3::CommandStatus

When a command is recieved from a master the application sends a code to indicate if it was successfull or if not what class of error was encountered.

Each code has a description that indicates its customary meaning.

Enumerator:
CS_SUCCESS 

command was successfully recieved and handled

CS_TIMEOUT 

command timedout before completing

CS_NO_SELECT 

command requires being selected before operate, configuration issue

CS_FORMAT_ERROR 

bad control code or timing values

CS_NOT_SUPPORTED 

command is not implemented

CS_ALREADY_ACTIVE 

command is allready in progress or its allready in that mode

CS_HARDWARE_ERROR 

something is stopping the command, often a local/remote interlock

CS_LOCAL 

the function goverened by the control is in local only control

CS_TOO_MANY_OPS 

the command has been done too often and has been throttled

CS_NOT_AUTHORIZED 

the command was rejected because the device denied it or an RTU intercepted it

CS_UNDEFINED 

10 to 126 are currently reserved
