Industrial strength collaboration

This is a commerically supported, rigorously tested, open source implementation of IEEE-1815 (dnp3) sponsored by Automatak. The core library is C++ that can target many server and embedded platforms. Java and .NET bindings are provided. It is licensed under commercially-friendly Apache License, Version 2.0. Please join the mailing list, introduce yourself, and post questions.

Like all implementations of the protocol, opendnp3 is not endorsed by dnp.org or IEEE. It is a seperate project that tracks the standard with a reference implementation. Automatak is a premium member of dnp.org and participates on the dnp technical committee.

Security

We take security testing very seriously. In the off chance that you find a security vulnerability in the library, please contact info@automatak.com and you will be provided with a secure way to disclose the details. Do not send any details of security issues to the mailing list or in plain text emails!

How long has this library been around?

It's a fork of this project with many enhancements, commercial support options, and a community of experts. The library has been in use since 2006 and has been deployed extensively across the industry since the open source release in 2010. The improvements in this project are based on almost 7 years of user feedback and real world deployment experience.

Why would I use this over the original project?

Many engineering hours have gone into improving the original codebase. Here's a short list:

  • Custom high-performance Java bindings
  • Multi-core support improves responsiveness and scalability
  • Extensive online documentation
  • Object model for managing channels, masters, and outstations
  • Masters and outstations independently handle all control functions and types
  • There is only one library with one namespace: opendnp3
  • Updated build systems (GNU autotools and Visual Studio)
  • Size reduction for embedded systems
  • Aggressive release cycle and compliance maintainence
